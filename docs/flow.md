# Жизненный цикл и Пользовательский интерфейс (UI)

Данный документ описывает основной процесс (flow) работы системы аукционов и особенности встроенного интерфейса управления.

## 1. Основной Flow (Процесс)

Система построена на событийной модели и взаимодействии в реальном времени.

### Шаг 1: Подготовка (CRUD)
Администратор или скрипт создает аукцион через HTTP API. На этом этапе задаются:
- Начальная цена.
- Шаг аукциона.
- Время начала и окончания.

### Шаг 2: Регистрация участников
Компании должны быть зарегистрированы как участники конкретного тендера через эндпоинт `/participate`. Без этого ставка через WebSocket будет отклонена.

### Шаг 3: Активация
Аукцион может быть запущен:
- Автоматически планировщиком (`Scheduler`) при наступлении `startAt`.
- Вручную через обновление статуса в API.

### Шаг 4: Торги (WebSocket)
1. Клиент подключается к `/ws/{tenderId}`.
2. Получает `snapshot` текущего состояния.
3. Отправляет сообщения `place_bid`.
4. Сервер валидирует ставку и рассылает `price_updated` всем участникам.

### Шаг 5: Завершение
По наступлении `endAt` или принудительно, сессия закрывается, фиксируется победитель, и брокер сообщений (RabbitMQ) уведомляет внешние системы.

## 2. Developer UI (Панель управления)

Интерфейс, доступный по адресу `/ui`, является **инструментом для разработчиков** и тестировщиков, а не конечным продуктом для пользователей.

### Назначение
UI служит для визуализации работы движка аукционов, отладки WebSocket-соединений и быстрой проверки бизнес-правил вручную.

### Ключевые функции

#### 1. CRUD Аукционов (Управление)
На главной странице UI представлен список всех аукционов в базе данных.
- **Создание**: Форма для быстрого добавления нового тендера с нужными параметрами.
- **Просмотр**: Список с фильтрацией по статусу.
- **Удаление/Редактирование**: Возможность оперативно изменить параметры или удалить тестовые данные.

#### 2. Демо-торги (Auction Detail)
При переходе в конкретный аукцион открывается панель торгов:
- **Логирование событий**: В реальном времени отображаются все входящие WebSocket-события (`snapshot`, `price_updated`, `error`).
- **Симуляция ставки**: Позволяет отправить ставку от имени любого UUID (если он зарегистрирован как участник).
- **Авто-подстановка**: UI автоматически предлагает следующую корректную цену (текущая минус шаг) для удобства тестирования.
- **Статус соединения**: Индикация живого подключения и автоматический реконнект.

### Режим разработки
В UI встроены инструменты для смены `personId` и `companyId` "на лету", чтобы можно было имитировать торги между разными участниками в разных вкладках браузера.
