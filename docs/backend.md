# Документация Backend (Go) — Сервис Аукционов

Данный документ описывает архитектуру, механизмы работы и бизнес-логику серверной части системы аукционов.

## 1. Архитектура проекта

Проект написан на языке Go с использованием принципов чистой архитектуры (Clean Architecture).

### Основные слои:
- **cmd/server/main.go**: Точка входа. Инициализация зависимостей (БД, AMQP, Logger), запуск HTTP-сервера и планировщика (Scheduler).
- **internal/http**: Слой транспорта. Обработка HTTP-запросов, маршрутизация (chi router) и управление WebSocket-соединениями.
- **internal/auction**: Ядро бизнес-логики.
    - `Manager`: Управляет жизненным циклом аукционных сессий.
    - `Session`: Активный процесс конкретного аукциона в памяти, обрабатывающий ставки и рассылающий события.
- **internal/db / internal/auction/repository**: Слой данных. Реализация работы с PostgreSQL через pgx.
- **internal/amqp**: Интеграция с брокером сообщений (RabbitMQ) для публикации событий о начале и завершении аукционов.
- **internal/scheduler**: Фоновый процесс, который запускает аукционы по расписанию.

## 2. Работа сокетов (WebSocket)

WebSocket используется для обеспечения реального времени (real-time) взаимодействия с участниками.

### Протокол подключения
- **URL**: `GET /ws/{tenderId}`
- При подключении сервер создает `Client` и подписывает его на события `Session`.
- Сразу после установления соединения клиент получает событие `snapshot` с текущим состоянием аукциона.

### Входящие сообщения (от клиента)
Клиент отправляет JSON для выполнения действий:
```json
{
  "type": "place_bid",
  "bid": 95000,
  "companyId": "comp-1",
  "personId": "user-1"
}
```

### Исходящие события (от сервера)
Сервер рассылает события всем подключенным участникам:
- `snapshot`: Полное состояние аукциона.
- `price_updated`: Уведомление о новой принятой ставке.
- `started`: Аукцион перешел в статус Active.
- `finished`: Аукцион завершен.

## 3. HTTP API (Routes)

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/auctions` | Создание нового аукциона |
| `GET` | `/auctions` | Список всех аукционов |
| `GET` | `/auctions/{tenderId}` | Получение информации о конкретном аукционе |
| `PATCH` | `/auctions/{tenderId}` | Обновление параметров аукциона |
| `DELETE` | `/auctions/{tenderId}` | Удаление аукциона |
| `POST` | `/auctions/{tenderId}/participate` | Регистрация участника (компании) в аукционе |
| `GET` | `/auctions/{tenderId}/bids` | История всех ставок по тендеру |
| `GET` | `/ws/{tenderId}` | WebSocket соединение |

## 4. Бизнес-логика аукциона

Аукцион работает по модели понижения цены (редукцион). Подробное описание всех правил и проверок вынесено в отдельный документ:

👉 **[Бизнес-правила и проверки](business-rules.md)**

### Краткий обзор:
1.  **Минорные единицы**: Все денежные значения (цены, шаги) передаются и хранятся в **минорных единицах** (`int64`).
2.  **Идентификаторы (UUID)**: Все ID тендеров, компаний и пользователей должны быть в формате **UUID**.
3.  **Статус Active**: Ставки принимаются только в активной фазе аукциона.
4.  **Правило цены**: Каждая новая ставка должна быть строго меньше текущей цены.
5.  **Шаг аукциона**: Соблюдение минимального шага и кратности.
6.  **Участники**: Обязательная предварительная регистрация.

### Жизненный цикл сессии:
1. `Manager` загружает аукцион из БД.
2. `Scheduler` или HTTP-запрос переводит его в статус `Active`.
3. Создается `Session` в оперативной памяти для быстрой обработки ставок.
4. При каждой валидной ставке:
    - Обновляется состояние в памяти.
    - Делается запись в БД (транзакционно).
    - Рассылается сообщение через WebSocket всем участникам.
5. По завершении статус меняется на `Finished`, сессия закрывается, результат фиксируется в БД.

## 5. Модели данных (JSON)

### Snapshot (Снимок состояния)
```json
{
  "tenderId": "T-123",
  "status": "Active",
  "startPrice": 100000,
  "step": 500,
  "currentPrice": 95000,
  "winnerId": "comp-1",
  "latestBid": {
    "companyId": "comp-1",
    "personId": "user-1",
    "bidAmount": 95000,
    "bidAt": "2024-03-21T10:00:00Z"
  },
  "startAt": "2024-03-21T09:00:00Z",
  "endAt": "2024-03-21T11:00:00Z"
}
```

### BidResult (Ответ на ставку)
```json
{
  "accepted": true,
  "currentPrice": 95000,
  "winnerId": "comp-1"
}
```
Если ставка отклонена, `accepted` будет `false` и заполнено поле `error`.
